# Phase 3.5 — UI Visual Polish & Interaction Layer

## Context and Problem Statement

After Phases 1–3 the engine works. Nodes render, halos fuse, gumballs grab, gizmos rotate, layout solves. But the app feels like a debug harness, not a designed product — every UX surface is panel-heavy, palettes are utilitarian dark/light, edges and labels render with no visual personality, and several structural integrations (Penumbra depth handoff most notably) are parked as "future work."

Phase 4 (valence orbits) ships a striking visual feature; landing it on a polished base means it reads well, while landing it on the current debug-harness base just adds noise. Phase 3.5 inserts visual polish + interaction-layer work between current state and Phase 4.

## Considered Options

The full design space and decision rationale lives in [`docs/research/2026-05-02-phase-3.5-ui-polish.md`](../research/2026-05-02-phase-3.5-ui-polish.md). This ADR records the locked-in decisions.

### Architecture changes

1. **5-stage render pipeline** (replaces current 2-stage):
   1. Penumbra ray-march → color + depth dual render target
   2. Three opaque pass (nodes, edges, grid, labels) — depth-tests against Penumbra depth
   3. Penumbra alpha composite — uses Penumbra depth via `gl_FragDepth` so nodes occlude correctly
   4. Bloom postprocess pass (selected + hovered nodes only)
   5. UI overlay (DOM panels, HUDs)

2. **Theme as first-class config object** ([ADR 0008](0008-theme-system.md)). Replace boolean `isLight` with parameterized `Theme` config carrying palette + motion + panel-style.

3. **Panel primitive** unifying Settings/Perf/Display/Plane/Snapshot panels into one component with: open/close transitions, mutually-exclusive activation, drag-to-move, theme-aware styling.

### Execution waves (revised)

- **Wave 1 (Foundation, DONE)**: theme system (ADR 0008), three ADRs written
- **Wave 2 (Structural, current)**:
  - 2a: Penumbra depth output (color + depth dual target) — ADR 0007 / Penumbra Bug 0030
  - 2b: Qualia depth-aware composite via custom ShaderMaterial — ADR 0007 / Qualia Bug 0023
  - 2c: Penumbra atmospheric render mode (coarse march → seed map → fog + particles) — Penumbra ADR 0009
  - 2d: Qualia atmospheric mode controls (mix slider, particle/fog density, step count) — Qualia ADR 0009
- **Wave 3 (Functional UX, animation-light)**: Cmd+K (C), HUD strip (F), zoom-to-cursor (G), FPS HUD (L)
- **Wave 4 (Functional visuals)**: Semantic zoom labels LOD 0/1/2 (B), Bug 0022 diagnostics
- **Wave 5 (Close-out)**: ADR close-out, log entries, verification snapshot

Phase 6 (deferred) holds: animated transitions (E), perlin sway (H), bloom (I), mutually-exclusive panels (J), radial menu (K), nav cube (M), edge hover glow (N).

### Consequences

- **Good:** the app reads as a designed product, not a debug harness. Monument palette gives instant visual identity. Cmd+K compresses panel discovery. Depth composite makes Penumbra a 3D presence.
- **Good:** theme refactor + panel primitive are foundations Phase 4 (valence orbits, electron animation) and beyond will build on.
- **Good:** structural changes (depth composite) that have been parked as "future work" through six bug rounds finally land — the same architectural bug isn't filed for the seventh time.
- **Bad:** ~22 hours of work spread across UI, renderer, and one Penumbra version bump. Significantly larger than past phases. Mitigated by wave-based shipping.
- **Bad:** five-stage render pipeline is more complex to debug. Mitigated by depth output being optional (PenumbraPass can skip the depth target if a host doesn't need it).
- **Bad:** Monument palette won't suit every user. Mitigated by keeping dark + light themes as alternatives.

## Implementation Notes

- Theme refactor (decision A + ADR 0008) is the foundation; ship first.
- Panel primitive (E + J) is the second foundation; ship next.
- Depth composite (D + ADRs 0007, Penumbra 0030) is the structural unlock; ship in Wave 2.
- All other decisions can ship in any order after Wave 1–2.
